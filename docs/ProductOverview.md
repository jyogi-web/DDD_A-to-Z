# ゲーム企画書 — Lang War

## 世界観

プログラミング言語が擬人化されたキャラクターとして存在する世界。各言語は「勢力」を持ち、現実のGitHub上での活動量がそのままゲームの勢力に反映される。コードを書けば書くほど、自分の推し言語が強くなる。

## ゲームの核心ルール

GitHubの特定言語でのCommit・PRが「勢力ポイント」に変換される。勢力ポイントは言語ごとに累積し、マップ上のテリトリーや順位に反映される。ポイントの重みづけ案は以下の通り。

| アクション | ポイント |
|---|---|
| Commit (1ファイル以上変更) | +1 |
| Pull Request (マージ済み) | +5 |
| Issue close (言語タグあり) | +2 |
| Star獲得 (言語リポジトリ) | +0.1 |

## 勝敗・サイクル

決めるべき論点が3つある。週次リセット制（毎週ゼロから争う緊張感）か永続蓄積制（歴史として積み上がる重厚感）か、あるいはシーズン制（両方のいいとこ取り）か。現時点では**シーズン制**が一番ゲームとして面白くなりそう。

## キャラクター設定（案）

各言語は能力値とキャラクター性を持つ。たたき台。

| 言語 | キャラ属性 | 強み能力値 |
|---|---|---|
| Rust | 鎧武者・無敵の守り | 安全性・速度 |
| Python | 魔法使い・万能型 | 汎用性・学習速度 |
| Go | 忍者・シンプル特化 | 並列処理・起動速度 |
| Java | 重装騎士・堅牢 | 安定性・エコシステム |
| TypeScript | 変身戦士・適応型 | フロント〜バック |
| Haskell | 賢者・理論派 | 型安全性 |
| Zig | 原始の戦士・最低レイヤー | メモリ制御 |

## UXイメージ

ログインするとマップ画面が表示され、各言語の勢力圏がビジュアルで見える。自分のGitHubアカウントを連携すると、自分の活動が即座に勢力に反映される。キャラクターをクリックするとその言語のステータスと最近の活動ログが見られる。
